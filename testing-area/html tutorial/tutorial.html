What is HTML?

HTML is the standard markup language for documents designed to be displayed in a web browser. A markup language is a text-encoding system which specifies the structure and formatting of a document. In layman's terms, it acts like the skeleton of a web page by telling the browser how to display the content. HTML markup consists of several components called tags. They commonly come in pairs as shown in the example below:

This is called an HTML Element. It consists of the opening(or start) tag, content and the closing(end tag) tag. Elements can also have attributes:

Attributes contain extra information about the element, it won't appear in the actual content. They always have a space between it and the element name, followed by an equal sign and the value is wrapped by opening and closing quotation marks.

How to write HTML?

A simple text editor is all that we need to start learning HTML.

HTML Documents

All HTML documents must start with a document type declaration: <!DOCTYPE html>. It represents the document type and helps browsers display web pages correctly. It MUST only appear once at the top of the page before any HTML tags. It is not case sensitive, which means you can write it in lowercase.

An example of a HTML5 document

<!DOCTYPE html>
<html>
<head>
<title>Page Title</title>
</head>
<body>

<h1>My First Heading</h1>
<p>My first paragraph.</p>

</body>
</html>

HTML Headings

HTML Headings are defined with the <h1> to <h6> tags. <h1> defines the most important heading, the <h6> the least important.

<h1>This is heading 1</h1>
<h2>This is heading 2</h2>
<h3>This is heading 3</h3>
<h4>This is heading 4</h4>
<h5>This is heading 5</h5>
<h6>This is heading 6</h6>

Note: Only use one <h1> per page. In addition to making our website more user-friendly, html headings can also help to improve our SEO. Search Engine Optimization is the process of improving our website to increase its visibility in Google and other search engines whenever users search for products, information, services etc.

The most common example of on-page SEO is optimizing a piece of content to a specific keyword. For example, We're making a HTML tutorial website, our keyword might be “html tutorial.” We'd include that keyword in our page's title, meta description, headers, and body.

HTML Paragraphs

HTML paragraphs are defined with the <p> tag. It defines a paragraph of text.

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

HTML Links

HTML links are defined with the <a> tag. The <a> tag (anchor tag) in HTML is used to create a hyperlink on the webpage. This hyperlink is used to link the webpage to other web pages or some section of the same web page. The link's destination is specified in the href attribute:

<a href="#">This is a link</a>

HTML Images

HTML images are defined with the <img> tag. It is used to embed an image in an HTML page. Images are not technically inserted into a web page; images are linked to web pages. The <img> tag creates a holding space for the referenced image. The <img> tag has two required attributes:

src - specifies the path to the image

alt - specifies an alternate text for the image, if the image for some reason cannot be displayed

Note: Also, always specify the width and height of an image. If width and height are not specified, the page might flicker while the image loads.

<img src="#" alt="alternate text" width="150" height="150" />

HTML Meta tags

The HTML document itself begins with <html> and ends with the </html> tag. The visible part of the HTML document is between <body> and </body>. The <head> element is a container for metadata (data about data) and is placed between the <html> tag and the <body> tag. Metadata is data about the HTML document, such as the document title, character set, styles, scripts, and other meta information. Metadata is not displayed.

Some examples of <meta> tags:

<meta name="keywords" content="HTML, CSS, JavaScript"> defines words for search engines.

<meta name="description" content="Tutorial for HTML and CSS"> defines a description of our webpage.

<meta name="author" content="Ivan and Aleksandar"> defines the author(s) of a page.

<meta http-equiv="refresh" content="30"> refreshes the document every 30 seconds.

<meta name="viewport" content="width=device-width, initial-scale=1.0"> sets the viewport to make our website look good on all devices.

The viewport is the user's visible area of a webpage. It varies with the device, it is smaller on a smartphone than on a tablet or computer screen. It gives the browser instuctions how to control the page's dimensions and scaling.

The width=device-width part makes the width of the page follow the width of our device.

The initial-scale=1.0 sets the initial zoom level when the page is first loaded by our browser.

Semantic Elements in HTML

The semantic code (SEM) points to any element in a text that suggests a particular, often additional meaning by way of connotation. Many websites contain HTML code like: <div id="nav"> <div class="header"> <div id="footer"> to indicate navigation, header, and footer. However, <div> and <span> elements are non-semantic. By themselves, they tell nothing about their content.

Here are some semantic elements that we can use to define different parts of a web page:

  • <article>
  • <aside>
  • <details>
  • <figcaption>
  • <figure>
  • <footer>
  • <header>
  • <main>
  • <nav>
  • <section>

A block of code that uses semantic elements:

<header></header>
<section>
<article>
<figure>
<img>
<figcaption></figcaption>
</figure>
</article>
</section>
<footer></footer>

VS a block of code that uses non-semantic elements:

<div id="header"></div>
<div class="section">
<div class="article">
<div class="figure">
<img>
<div class="figcaption"></div>
</div>
</div>
</div>
<div id="footer"></div>

What are the big differences?

The first line of code is much easier to read! Imagine we have to read through hundreds or thousands of lines of code? The easier it is to read and understand that code, it makes our job easier!

Secondly, it has greater accessibity. Search engines and assistive technologies like screen readers are also able to better understand the context and content of our webiste.

Overall, semantic elements also lead to more consistent code.

Let's break down the elements

The <header> element is generally found at the top of a document, a section, or an article and usually contains the main heading and some navigation and search tools.

The <nav> element defines a set of navigation links. The <nav> element is intended only for major blocks of navigation links.

The <aside> element defines some content aside from the content it is placed in (like a sidebar) and it should be indirectly related to the surrounding content.

<figure> is for wrapping your image content around it, and <figcaption> is to caption your image.

A <footer> is generally found at the bottom of a document, a section, or an article. Just like the <header> the content is generally metainformation, such as author details, legal information, and/or links to related information.

The <section> element defines a section in a document. A section is a thematic grouping of content, typically with a heading, such as Chapters, Introduction, News items, Contact information.

The <article> element specifies independent, self-contained content. It can be used for forum posts, blogs, user comments, product cards, newspaper articles.

HTML Forms

<form> is an HTML element to collect input data containing interactive controls. <form> is a container that contains input elements like text, email, number, radio buttons, checkboxes, submit buttons.

<form>
form elements...
</form>

The <form> element can contain one or more elements, such as <input>, <textarea>, <button>, <select>, <option>.

An HTML Form




Have you eaten today?



Let's break down the example above

<form>
<label for="fname">Name:</label>
<input type="text" id="fname" name="fname" value="Tom">
<label for="species">Select your species:</label>
<select name="species" id="species">
<option value="cat">Cat</option>
<option value="dog">Dog</option>
</select>
<p>Have you eaten today?</p>
<input type="radio" id="yes" name="answer" value="yes">
<label for="yes">Yes</label>
<input type="radio" id="no" name="answer" value="no">
<label for="no">No</label>
<input type="submit" value="Submit">
</form>

The <label> tag is useful for screen-readers and helps users who have difficulty clicking on small regions like buttons and checkboxes. The for attribute of the <label> tag shouyld be equal to the id attribute of the <input> element to bind the two together.

The <input type="text"> defines a single-line input field for text input. We added the value attribute so it would display "Tom".

The <select> tag is used to create a drop-down list. The id attribute is needed to associate the drop-down list with a label. The <option> tags inside the <select> element define the available options in the drop-down list. If we have a long list of options, we can group related options within the <optgroup> tag!

The <input type="radio"> defines a radio button. Radio buttons let us select only one of a limited number of choices. For one or more options of a limited number of choices, we will use <input type="checkbox"> instead of radio.

The <input type="submit"> defines a button for submitting the form data to a form-handler.

Each input field must have a name attribute to be submitted. If it is omitted, the value of the input field will not be sent at all.

If we want a field with a multi-line text input control, we will add the <textarea> tag. It is useful for inputs like comments and reviews. It can hold an unlimited number of characters. The size of a text area is specified by the cols and rows attributes (or with CSS).

The <button> element defines a clickable button. We must always specify the type attribute for the button element!

Note: We should always add the <label> tag for best accessibity practices.

HTML Input Attributes

The value attribute specifies an initial value for an input field.

The readonly attribute specifies that an input field is read-only. It can't be modified.

The disabled attribute specifies that an input field should be disabled. It is unusable and un-clickable. This value will not be sent when submitting the form.

The size attribute specifies the visible width, in characters, of an input field. The default value is 20. This attribute works with input types: text, search, url, email and password.

The min and max attributes specify the minimum and maximum values for an input field. They work with: number, range, date, month, time and week. maxlength attribute specifies the maximum number of characters allowed in an input field.

The multiple attribute specifies that the user is allowed to enter more than one value in an input field. It works with email and file.

The placeholder attribute specifies a short hint that describes the expected value of an input field. It is displayed in the input field before we enter a value. It works with text, search, url, email, password.

The required attribute specifies that an input field must be filled out before submitting the form.

The autofocus attribute specifies that an input field should automatically get focus when the page loads.
